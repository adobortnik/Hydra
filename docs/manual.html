📖 Hydra User Manual

Step-by-step workflows for running your Instagram phone farm

v1.0 — February 2026

🏁 Getting Started

Welcome to Hydra! This manual walks you through every workflow you'll need — from connecting your first phone to running a full 50-device farm. If you're looking for technical API details, check the API Reference instead.

First Time Setup Checklist

Before you do anything, make sure you have all of these ready:

  • Android phones — Samsung devices connected to the same WiFi network as the server
  • ADB enabled — Developer Options → USB Debugging turned on for each phone
  • Instagram clone APKs — Clone apps installed (androie through androip — up to 12 clones per device)
  • Instagram accounts — A list of accounts in username:password or username:password:2fa_secret format
  • Server running — The Hydra dashboard server is up and accessible
  • Network — All phones and the server on the same LAN (10.1.11.x subnet)

Accessing the Dashboard

Open your browser and go to:

http://10.1.11.168:5055

This is your control center for everything. The dashboard works on any device — desktop, tablet, or phone. No login is required on the local network.

📱 Adding New Devices

Every phone in your farm connects to the server over WiFi using ADB (Android Debug Bridge). Here's how to get a new phone online.

How to Connect a Phone (ADB over WiFi)

Enable Developer Options

On the phone, go to Settings → About Phone and tap Build Number 7 times. You'll see "Developer mode enabled."

Enable USB Debugging

Go to Settings → Developer Options and turn on USB Debugging. Also enable Wireless Debugging if available (Android 11+).

Connect via USB first

Plug the phone into the server via USB cable. Accept the "Allow USB debugging" prompt on the phone. Run adb devices to verify it shows up.

Switch to WiFi mode

Run adb tcpip 5555 to switch the phone to WiFi debugging mode. Then disconnect the USB cable.

Connect over WiFi

Find the phone's IP address (Settings → WiFi → tap the connected network). Then run adb connect 10.1.11.XXX:5555. You should see "connected to 10.1.11.XXX:5555".

💡 Pro tip

Assign static IP addresses to each phone through your router's DHCP settings. This way phones always get the same IP after a restart, and ADB reconnects automatically.

How Devices Appear in Device Manager

Once a phone is connected via ADB, it automatically shows up in the Device Manager page (/devices). You'll see:

  • Device name — the phone's model name (e.g., "Samsung Galaxy A13")
  • IP address — the WiFi IP address used for ADB
  • Status — 🟢 Online or 🔴 Offline
  • Accounts — how many Instagram accounts are assigned to this device
  • Clone apps — which Instagram clones are installed
[Screenshot: Device Manager showing a list of connected phones with status indicators]

Device Groups

Device groups let you organize your phones logically. Common groupings:

  • By mother account — all phones working for the same mother
  • By location — phones on different shelves or racks
  • By purpose — follow bots vs. engagement bots vs. content posters

To assign a device to a group, go to Device Manager → click a device → Edit → Group. Groups are useful for bulk actions — you can start/stop all bots in a group, or apply settings to an entire group at once.

👥 Adding & Managing Accounts

Accounts are the core of Hydra. Each Instagram account runs on a specific clone app on a specific device.

Importing Accounts

The fastest way to add accounts is bulk import. Go to Accounts → Import and paste your accounts in one of these formats:

username:password
username:password:2fa_secret

You can import hundreds of accounts at once. Each line is one account. The 2FA secret is the TOTP seed (the code you'd normally scan as a QR code) — Hydra uses it to automatically generate 2FA codes during login.

ℹ️ Import formats

Hydra also supports importing from CSV files and other formats via the Import V2 system. Check the API Reference for advanced import options.

Assigning Accounts to Devices

After import, accounts need to be assigned to a specific device and clone app. This tells Hydra which phone to use for each account.

Go to the Accounts page

Navigate to /accounts. You'll see all imported accounts, likely with status "pending_login".

Select accounts to assign

Use the checkboxes to select one or more accounts. Click Assign to Device.

Pick a device and clone slot

Choose the target device from the dropdown, then select which clone app slot (androie, androif, ... androip) to use. Each clone is a separate Instagram app installation.

Clone Apps — What They Are

Each phone can run up to 12 Instagram accounts simultaneously using clone apps. These are renamed copies of Instagram, each with its own data storage:

Clone Name Package Suffix Slot

Each clone app functions as a completely independent Instagram installation. They don't share cookies, sessions, or data. From Instagram's perspective, each clone looks like a different person on a different phone.

Account Statuses

Every account has a status that tells you where it is in its lifecycle:

Status Meaning What to Do
pending_login Imported but not logged in yet Run Login Automation
active Logged in and ready to run Good to go! Start the bot
challenge_required Instagram asked for verification Check email/phone verification, retry login
login_failed Login attempt failed Check credentials, try again
banned Account suspended by Instagram Appeal or replace the account
cooldown Temporarily rate-limited Wait it out (usually 24-48 hours)

🔐 Login Flow

Once accounts are imported and assigned to devices, you need to log them into Instagram. Hydra's Login Automation V2 handles this automatically.

Login Automation V2 — Step by Step

Go to Login Automation

Navigate to the Login Automation page from the dashboard sidebar. You'll see all accounts with their current login status.

Select accounts to log in

Check the accounts you want to log in. You can filter by status (e.g., show only "pending_login") to find accounts that need attention.

Click "Start Login"

Hydra will queue the selected accounts and begin logging them in one by one on their assigned devices. The automation opens Instagram on the phone, enters credentials, and handles any prompts.

Monitor progress

Watch the live status updates. Each account will transition from "pending_login" → "logging_in" → "active" (success) or "login_failed" / "challenge_required" (needs attention).

[Screenshot: Login Automation V2 page showing accounts being logged in with status indicators]

2FA Handling

If you imported accounts with 2FA secrets (the username:password:2fa_secret format), Hydra automatically generates the 6-digit code during login. No manual intervention needed.

If an account has 2FA enabled but you didn't provide the secret, you'll need to:

  1. Watch for the 2FA prompt on the login page
  2. Manually enter the code from your authenticator app
  3. Or add the 2FA secret to the account and retry
⚠️ Always save 2FA secrets

When setting up 2FA on accounts, always save the TOTP secret key. Without it, Hydra can't auto-generate codes and you'll need to manually handle every login.

What to Do When Login Fails

Problem Cause Solution
Wrong password Credentials are incorrect Update the password in account settings and retry
Challenge required Instagram wants email/SMS verification Complete the challenge manually on the device, or use a fresh account
Too many attempts Instagram rate-limited the login Wait 1-2 hours and try again. Don't spam retries.
Account disabled Instagram banned the account Try appealing through the app, or replace the account
Device offline Phone lost WiFi or ADB connection Reconnect the device first (see Adding Devices)

Bulk Login for Multiple Accounts

When you have dozens (or hundreds) of accounts to log in:

  1. Select all pending accounts — use the "Select All" checkbox or filter by status
  2. Click "Start Bulk Login" — Hydra will queue them all
  3. Let it run — the system processes accounts sequentially per device (one at a time per phone to avoid conflicts)
  4. Check back later — bulk login can take a while. Come back and review which accounts succeeded and which need manual attention
💡 Patience with bulk login

With 50 devices and 12 accounts each, a full login run can take several hours. Hydra adds small delays between logins on the same device to avoid Instagram flagging rapid sequential logins from the same phone.

⚙️ Setting Up the Bot

Before running the bot, you need to configure what it does. This is where bot settings, sources, and tags come in.

Bot Settings (Per Account)

Go to Bot Settings (/bot-settings) to configure each account's automation behavior. Key settings:

Time Windows

Define when the bot is active. Example: 08:00 - 23:00 means the bot only runs during these hours (local time). Outside this window, the account sleeps.

Actions

Choose which actions the bot performs:

  • Follow — follow users from your source lists
  • Like — like posts from feeds, hashtags, or explore
  • Comment — leave comments (uses your comment templates)
  • Watch Reels — watch reels to look active
  • Share to Story — share posts to the account's story
  • DM — send direct messages to users
  • Unfollow — unfollow users that were followed previously (after X days)

Delays & Limits

  • Min/Max delay between actions — random delay between each action (e.g., 30-90 seconds). Mimics human behavior.
  • Daily action limits — max follows, likes, comments per day. Stay under Instagram's radar.
  • Session breaks — periodic breaks where the bot pauses for a few minutes. Looks more natural.
⚠️ Start conservative

New accounts should start with very low limits (10-20 follows/day) and gradually increase over 2-4 weeks. See Tips & Best Practices for safe warmup schedules.

[Screenshot: Bot Settings page showing action types, delays, and limits configuration]

Sources — Where to Find People

Sources tell the bot where to find users to interact with. Go to Sources & Targets (/sources).

Types of Sources

  • Competitor followers — followers of a specific Instagram account (great for targeting a niche audience)
  • Hashtag feeds — users who post with specific hashtags
  • Location feeds — users posting from specific locations
  • Explore page — discover new users through Instagram's algorithm

How to Add a Source

Go to Sources & Targets

Open the Sources page from the sidebar.

Click "Add Source"

Choose the source type (competitor, hashtag, location) and enter the target.

Assign to accounts

Select which accounts should use this source. You can assign one source to many accounts or give each account different sources.

Tags — Organizing and Dedup

Tags let you categorize accounts and — more importantly — avoid duplicate interactions.

What is Tag Dedup?

When multiple slave accounts interact with the same target audience, there's a risk of the same user being followed by 10 different slaves. That looks suspicious. Tag dedup prevents this:

  • Accounts with the same tag share a dedup list
  • If Account A follows @user123, Account B (same tag) will skip @user123
  • This spreads your interactions across a wider audience instead of piling on the same people
💡 Tag strategy

Use one tag per mother account. All slaves promoting the same mother should share a tag. This ensures each target user only gets one follow/like from your farm, making everything look natural.

Mother/Slave Concept

This is the core of Hydra's strategy:

  • Mother account — your main Instagram account that you want to grow. This is the "real" account with your brand, content, and followers.
  • Slave accounts — worker accounts that interact with people on behalf of the mother. Their bios point to the mother, they share the mother's content, and their activity drives traffic to the mother.

A typical setup: 1 mother + 50-100 slaves. The slaves do all the risky automation work while the mother stays clean and grows organically from the attention.

▶️ Running the Bot

Once your accounts are logged in, settings configured, and sources added — it's time to run the farm.

Starting the Farm

Launch from Dashboard

Go to the dashboard and click "Start Farm". This starts the bot process for all active accounts on all connected devices.

Bots initialize

Each account's bot checks its settings, loads its sources, and connects to its assigned device/clone app. You'll see status changes in real time.

Action cycle begins

The bot picks an action (follow, like, etc.) based on its settings, executes it on the phone, waits the configured delay, then picks the next action. This repeats until the daily limit is reached or the time window closes.

What Happens During a Bot Cycle

Here's what a typical bot cycle looks like for one account:

  1. Wake up — check if it's within the active time window
  2. Pick a source — select a random source from the account's source list
  3. Find a target user — browse the source (hashtag feed, competitor followers, etc.) and find a user that hasn't been interacted with before
  4. Perform actions — follow the user, like 1-3 of their posts, maybe leave a comment
  5. Wait — random delay (e.g., 45-120 seconds)
  6. Repeat — go back to step 2 until daily limits are reached
  7. Sleep — take a session break or end for the day

Monitoring Your Bots

While the farm is running, you can monitor everything from the dashboard:

  • Dashboard Home — live overview of all running bots, today's action counts
  • Farm Stats (/farm-stats) — detailed charts and graphs of actions over time
  • Accounts page — check individual account status and last action time
  • Logs — check phone-farm/logs/ for detailed bot activity logs
[Screenshot: Dashboard home showing live bot activity — actions today, accounts running, recent actions feed]

Stopping the Farm

To stop all bots:

  • Dashboard — click "Stop Farm" button
  • Terminal — press Ctrl+C in the terminal running launch_farm.py
  • Individual accounts — you can stop/start specific accounts from the Accounts page
ℹ️ Graceful shutdown

When you stop the farm, bots finish their current action before shutting down. This avoids leaving Instagram in a weird state mid-action.

👤 Profile Automation

Profile Automation lets you set up your slave accounts' identities in bulk — usernames, bios, profile pictures, and more. Go to Profile Automation (/profile-automation).

Mother Account Clones

This mode copies a mother account's "style" to its slaves:

  • Similar bio structure with mentions of the mother account
  • Related profile themes
  • Links pointing back to the mother

Use this when you want slaves to be obviously connected to the mother — like fan pages or promotional accounts.

Select a mother account

In Profile Automation, choose the mother account you want to clone from.

Select slave accounts

Pick which slaves should receive the mother's style. You can select by tag, device group, or individually.

Configure clone settings

Choose what to clone: bio template, profile picture style, username pattern, link in bio.

Execute changes

Click "Apply" and Hydra will push the profile changes to each slave account on their respective devices.

Unique Personas

This mode generates completely unique, realistic identities for each slave account. Ideal when you want slaves to look like real, unrelated people.

What gets generated:

  • Usernames — natural-looking patterns (e.g., "sarah.miller.23", "jake_adventures", "emily.creates")
  • Bios — AI-generated bios that sound like real people wrote them
  • Profile pictures — unique images from the profile picture library
  • Full names — realistic first + last name combinations
💡 AI-powered bios

Hydra uses AI to generate bios that sound natural and unique. No two accounts will have the same bio. You can set the tone (casual, professional, funny) and include specific keywords.

Executing Profile Changes

After configuring profiles (either clones or unique personas):

  1. Review the preview — each account shows what will change
  2. Click "Execute Changes"
  3. Hydra pushes updates to each phone via ADB, opening Instagram and changing profile settings
  4. Progress shows in real time — you'll see each account getting updated
⚠️ Don't rush profile changes

Changing too many profile elements at once on a new account can trigger Instagram's spam detection. Change username first, wait a day, then update bio and picture.

📋 Job Orders

Job Orders let you create specific, targeted tasks and distribute them across your farm. Instead of general "follow from source" automation, jobs target specific users or content.

Creating a Follow Job

Go to Job Orders

Navigate to Job Orders (/job-orders) and click "New Job".

Choose action type: Follow

Select "Follow" as the job type.

Add target users

Paste a list of usernames to follow (one per line), or specify a source account to scrape followers from.

Assign workers

Choose which slave accounts should work on this job. You can pick individual accounts, tags, or device groups.

Set distribution

Choose how targets are split — evenly distributed, or each worker gets all targets. Set daily limits per worker.

Launch the job

Click "Create Job" and the system starts distributing work to the assigned accounts.

Creating a Like/Comment/Report Job

Same workflow as follow jobs, but you choose a different action type:

  • Like job — targets are post URLs or users whose posts to like
  • Comment job — targets are post URLs + you provide comment templates
  • Report job — targets are accounts or posts to report (use responsibly)

Monitoring Job Progress

Active jobs show in the Job Orders page with:

  • Progress bar — how many targets have been processed
  • Workers active — how many accounts are currently working on the job
  • Success/fail count — how many actions succeeded vs. failed
  • Estimated time remaining
[Screenshot: Job Orders page showing an active follow job with progress bar and stats]

Quick Mass Report

For mass reporting a specific account or post:

  1. Go to Job Orders → "Quick Mass Report"
  2. Enter the target account username or post URL
  3. Select the report reason
  4. Choose how many accounts to use
  5. Click "Launch" — reports are spread across your farm with natural delays
🚨 Use reporting responsibly

Mass reporting is a powerful tool. Only use it for legitimate purposes. Abuse can result in your own accounts being flagged by Instagram.

📸 Content Management

Hydra can automatically post content (photos, reels, stories) to your accounts. Go to Content & Media (/content).

Scheduling Posts, Reels & Stories

Upload media

Upload photos, videos, or story content to the content library. Supported formats: JPG, PNG, MP4.

Create a scheduled post

Select the media, write a caption (or use AI-generated captions), choose the post type (feed post, reel, story), and set the date/time.

Assign to accounts

Choose which accounts should post this content. You can post the same content to multiple accounts or create unique variations.

Let it go

At the scheduled time, Hydra opens Instagram on the assigned device, navigates to the posting flow, and publishes the content.

Content Library

The content library stores all your media files. You can organize them into folders, tag them, and reuse them across multiple accounts and schedules.

How Content Gets Posted

Hydra uses ADB to automate the phone's UI:

  1. Opens Instagram on the correct clone app
  2. Pushes the media file to the phone via ADB
  3. Navigates through the posting flow (new post → select media → add caption → share)
  4. Verifies the post was published successfully
ℹ️ Content tips

Vary your content across accounts. Posting the exact same photo from 50 accounts looks suspicious. Use different crops, filters, or captions for each account.

🔧 Troubleshooting

Things go wrong sometimes. Here are the most common issues and how to fix them.

"Database is locked"

This happens when multiple processes try to write to the database at the same time.

⚠️ Fix: Database is locked

Quick fix: Restart the dashboard server. This releases the database lock.

If it keeps happening: Restart the server PC or contact support.

Bot Not Starting

Symptom Cause Fix
Bot shows "idle" but should be running Account not assigned to a device Assign the account to a device + clone slot
Bot starts but immediately stops Outside the configured time window Check bot settings → time window
Bot shows errors in logs Device is offline or ADB disconnected Reconnect the device (see Adding Devices)
No sources configured Bot doesn't know where to find users Add at least one source in Sources & Targets
Daily limit reached Account already did its max actions today Wait until tomorrow or increase limits (carefully)

Device Not Connecting

Check WiFi

Make sure the phone is connected to the same WiFi network as the server. Try pinging the phone's IP: ping 10.1.11.XXX

Reconnect ADB

Run adb disconnect 10.1.11.XXX:5555 then adb connect 10.1.11.XXX:5555. If that fails, plug in the USB cable and run adb tcpip 5555 again.

Restart ADB server

If multiple devices are having issues: adb kill-server then adb start-server. Then reconnect all devices.

Reboot the phone

If nothing else works, reboot the phone and start the connection process from scratch. Check that Developer Options and USB Debugging are still enabled after reboot.

Account Got Banned — What to Do

  1. Don't panic — individual account bans are normal. It happens.
  2. Mark it as banned in Hydra — this stops the bot from trying to use it
  3. Try to appeal — open Instagram on the device, follow the appeal process. Some bans are reversible.
  4. If the appeal fails — remove the account from its device slot and assign a new account
  5. Investigate — check if many accounts got banned at once. If so, you might be running too aggressive. Dial back your settings.
🚨 Mass bans = reduce aggression

If more than 5 accounts get banned in a single day, immediately pause all bots and review your settings. Lower daily limits, increase delays, and check if you're hitting the same targets too hard.

Other Common Issues

Issue Fix
Dashboard not loading Check if the server is running. Restart the dashboard if needed.
Actions counted but nothing happening The phone screen might be off. Enable "Stay Awake" in Developer Options.
Clone app crashing Clear the clone app's cache: Settings → Apps → [clone] → Clear Cache. Reinstall if needed.
Photos not posting Check storage on the phone. Media transfer may fail if the phone is full.
Slow performance Reboot phones periodically. Android gets sluggish after weeks of continuous use.

💡 Tips & Best Practices

Years of running Instagram farms distilled into actionable advice. Follow these to maximize growth and minimize bans.

Warming Up New Accounts

New accounts need a "warmup" period before they can handle full automation. Instagram watches new accounts closely.

Day Follows/Day Likes/Day Comments/Day Notes
Day 1-3 5-10 10-20 0-3 Just browse, like a few things. Look human.
Day 4-7 10-20 20-40 3-5 Start light following. Watch some reels.
Week 2 20-40 40-80 5-10 Gradually increasing. Post 1-2 photos.
Week 3-4 40-80 80-150 10-20 Getting closer to full speed. Add stories.
Month 2+ 80-150 150-300 20-40 Full automation. Monitor for any issues.
💡 The golden rule

If in doubt, do less. It's always better to be conservative and avoid a ban than to push limits and lose accounts. Accounts are your most valuable resource.

Safe Daily Limits

These are the generally safe daily maximums for aged, warmed-up accounts:

  • Follows: 100-150/day (spread over 8-12 hours)
  • Unfollows: 100-150/day
  • Likes: 200-300/day
  • Comments: 30-50/day
  • DMs: 20-40/day
  • Story views: 200-500/day
⚠️ Limits change

Instagram constantly adjusts their detection algorithms. What's safe today might be too aggressive next month. Monitor ban rates and adjust accordingly. When in doubt, pull back 20%.

Avoiding Bans

  • Use realistic delays — 30-120 seconds between actions. No human scrolls and likes at machine speed.
  • Vary your actions — don't just follow 100 people in a row. Mix follows with likes, comments, and story views.
  • Take breaks — configure session breaks every 30-60 minutes. Real people don't use Instagram for 12 hours straight.
  • Post content — accounts that only follow/like but never post look suspicious. Post occasionally.
  • Complete profiles — accounts with no profile picture, no bio, and no posts get flagged faster.
  • Don't change IP addresses — keep phones on the same WiFi network. IP changes trigger reviews.
  • Warm up new accounts — always follow the warmup schedule above.
  • Unfollow gradually — don't unfollow 200 people at once. Spread it out over days.

Tag Strategy for Dedup

Smart tagging is crucial for large farms. Here's the recommended approach:

One Tag Per Mother Account

If you're growing 3 mother accounts, create 3 tags. Assign all slaves for Mother A to "tag_mother_a", all slaves for Mother B to "tag_mother_b", etc.

Why This Matters

  • Without dedup: User @john gets followed by 15 of your slaves in one day. Instagram notices. @john reports them. Mass ban.
  • With dedup: User @john gets followed by 1 slave. That's it. All other slaves in the same tag skip @john. Natural.

Cross-Tag Interactions

Accounts with different tags can interact with the same users. This is fine — if Mother A's slave and Mother B's slave both follow @john, that's two different "people" from two different contexts. Not suspicious.

💡 Advanced tip: Sub-tags

For very large farms (200+ slaves per mother), split slaves into sub-groups with slightly different sources. This gives each sub-group a unique audience segment while still sharing the dedup list through tags.

General Tips

  • Reboot phones weekly — prevents memory leaks and Android slowdowns
  • Keep the dashboard open — check it at least twice daily to catch issues early
  • Back up your data — your database is your lifeline. Back it up regularly.
  • Monitor Farm Stats — if daily actions suddenly drop, something is wrong. Investigate immediately.
  • Rotate comment templates — stale comments get flagged. Update them regularly.
  • Don't run all accounts at the same time — stagger start times to avoid a spike of activity from the same IP range
