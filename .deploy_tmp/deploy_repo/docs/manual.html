📖 Hydra User Manual

How to use the dashboard — step by step

v1.0 — February 2026

🏁 Getting Started

This manual shows you how to use every part of the Hydra dashboard. If you're looking for a deeper technical reference, see the Technical Reference.

Before You Begin

Make sure you've completed the initial setup — your admin should have given you:

  • Dashboard URL — the address to open Hydra in your browser
  • Phones connected — devices should already appear in Device Manager
  • Clone apps installed — Instagram clones ready on the phones
  • Accounts prepared — a list of Instagram accounts to import

Opening the Dashboard

Open the dashboard URL in any browser (Chrome recommended). No installation needed — it works on desktop, tablet, and mobile.

[Screenshot: Dashboard home page after first login]

📱 Adding New Devices

Devices are the phones in your farm. Each phone can run multiple Instagram accounts using clone apps.

How a New Phone Appears

Once a phone has been connected to the network and set up by your admin, it automatically shows up in Device Manager. You don't need to add it manually.

[Screenshot: Device Manager — list of phones with status indicators]

What You'll See for Each Device

  • Device name — the phone model (e.g. "Samsung Galaxy A13")
  • Status indicator — 🟢 Online or 🔴 Offline
  • Assigned accounts — how many accounts are running on this phone
  • Clone slots — which Instagram clone apps are installed and available
ℹ️ Phone not showing up?

If a newly connected phone doesn't appear, contact your admin — the device may need to be connected to the network or have its debugging enabled.

Device Groups

You can organize phones into groups for easier management. Common groupings:

  • By mother account — all phones working to grow the same mother
  • By task type — follow bots vs. engagement bots vs. content posters
  • By rack or location — physical organization

To assign a group: click any device in Device Manager → Edit → Group. Groups let you apply bulk actions to all devices in the group at once.

👥 Adding & Managing Accounts

Every Instagram account you run lives in the Accounts page. This is where you import, assign, and monitor them.

Importing Accounts

Open the Import page

Go to Accounts → Import in the sidebar.

Paste your accounts

Enter one account per line, using this format:

username:password
username:password:2fa_secret

The 2FA secret is optional — it's the TOTP key that lets Hydra automatically generate verification codes during login. If you have it, include it.

Click Import

Hydra validates the format and adds all accounts. They'll appear with status pending_login — ready to be logged in.

💡 Bulk imports

You can paste hundreds of accounts at once. Hydra handles duplicates — if an account already exists, it'll be skipped.

Assigning Accounts to Devices

After import, each account needs to be placed on a specific phone and clone slot.

Select accounts

In the Accounts page, check the boxes next to the accounts you want to assign.

Click "Assign to Device"

A dialog opens. Choose the target device from the dropdown and the clone slot to place the account in.

Confirm

The account is now linked to that phone and clone. It's ready for login.

Clone Slots — One Account Per Slot

Each phone has up to 12 clone slots (named androie through androip). Each slot is a separate Instagram app installation — completely independent from the others. One account goes in one slot.

Account Statuses

Every account has a colour-coded status badge. Here's what each one means:

Status What It Means What to Do
pending_login Imported, not logged in yet Run login automation
active Logged in and running Nothing — it's working
challenge_required Instagram wants extra verification See Login Fails below
login_failed Login didn't work Check credentials, retry
banned Account suspended by Instagram Appeal or replace
cooldown Temporarily rate-limited Wait 24-48h, it resolves on its own

🔐 Login Flow

After importing and assigning accounts, the next step is logging them into Instagram.

How to Log In Accounts

Open Login Automation

Go to Login Automation in the sidebar. You'll see all accounts and their current login status.

Filter by status

Use the status filter to show only pending_login accounts — these are the ones that need to be logged in.

Select accounts

Check the accounts you want to log in. Use "Select All" for the whole batch.

Click "Start Login"

Hydra begins logging in the selected accounts on their assigned devices. Progress updates appear in real time.

Monitor results

Watch as accounts transition: pending_login → active (success) or login_failed / challenge_required (needs attention).

[Screenshot: Login Automation page — accounts being logged in with live status updates]

2FA Handling

If you included the 2FA secret during import (user:pass:2fa_secret), Hydra generates the verification code automatically. No manual work needed.

If an account has 2FA enabled but you didn't provide the secret, you'll need to enter the code manually when prompted.

⚠️ Always save your 2FA secrets

When setting up 2FA on accounts, save the TOTP secret key. Without it, every login requires manual code entry.

When Login Fails

Problem What to Do
Wrong password Go to Accounts, click the account, update the password, then retry login
Challenge required Instagram wants email or SMS verification. You may need to verify manually on the device, then retry.
Too many attempts Wait 1-2 hours before retrying. Don't spam the login button.
Account disabled The account is banned. Try appealing through Instagram, or replace it with a new account.
Device offline The phone isn't connected. Check Device Manager — if it's 🔴 Offline, contact your admin.

Bulk Login Tips

  • You can select all pending accounts and login them in one batch
  • Hydra processes one account at a time per device to avoid conflicts
  • Large batches (100+ accounts) take time — check back periodically for results
  • Failed logins can be retried by selecting them and clicking "Start Login" again

⚙️ Setting Up the Bot

Before running your accounts, configure what they should do. All settings live in the Bot Settings page.

Configuring an Account

Go to Bot Settings and select an account (or select multiple to edit in bulk). You'll see these setting groups:

Time Window

Set the hours when this account should be active. Example: 08:00–23:00. Outside this window, the account stays idle — just like a real person sleeping.

Actions

Toggle which actions the account should perform:

Action What It Does
FollowFollow users from your source lists
LikeLike posts from feeds, hashtags, or explore
CommentLeave comments using your templates
Watch ReelsWatch reels in the feed
Share to StoryShare posts to the account's story
DMSend direct messages
UnfollowUnfollow users after a set number of days

Delays

Set the min/max delay between actions (in seconds). Hydra picks a random time in this range before each action. Example: 30–90 seconds. This makes activity look natural.

Daily Limits

Maximum number of each action per day. Once a limit is reached, that action stops for the rest of the day. Set conservative limits for new accounts (see Tips).

⚠️ Start low, ramp up slowly

New accounts should begin with very low limits (10-20 follows/day) and increase gradually over weeks. Jumping straight to high limits is the fastest way to get banned.

[Screenshot: Bot Settings page — toggles for actions, delay sliders, limit fields]

Sources — Where Accounts Find People

Sources tell your accounts where to find users to interact with. Open the Sources & Targets page.

Adding a Source

Click "Add Source"

Choose the type: Competitor (followers of an account), Hashtag, or Location.

Enter the target

Type in the competitor username, hashtag, or location name.

Assign to accounts

Pick which accounts should use this source. You can assign one source to many accounts.

💡 Pick relevant sources

The best sources are competitor accounts whose followers match your target audience. The more relevant the source, the more likely people will follow your mother account back.

Tags & Dedup

Tags group accounts together for deduplication — preventing multiple accounts from interacting with the same person.

How it works:

  • Accounts with the same tag share a "seen" list
  • If Account A follows @someone, all other accounts with the same tag will skip @someone
  • This spreads your reach wider and avoids the suspicious pattern of one person being followed by many of your accounts

Recommended setup:

Create one tag per mother account. Assign all slave accounts promoting the same mother to the same tag. This way every target user gets interacted with by at most one of your accounts.

Mother / Slave Setup

The core Hydra concept:

  • Mother — your main account that you want to grow. This is your brand.
  • Slaves — worker accounts that interact with people and point them toward the mother (via bio links, story shares, tags).

A typical setup: 1 mother + 50–100 slaves. Configure which account is the mother in Bot Settings → Mother Account.

▶️ Running the Bot

Accounts are logged in, settings configured, sources added — time to start the farm.

Starting

Go to Dashboard Home

The main dashboard page has a "Start Farm" button. Click it to launch all active accounts.

Watch accounts go live

Active accounts begin their daily routine. The dashboard shows real-time counts: actions performed, accounts running, any errors.

You can also start/stop individual accounts from the Accounts page — just click an account and toggle its run state.

What to Watch While Running

Once the farm is going, keep an eye on these:

📊

Dashboard Home

Quick glance — active bots, today's actions, any errors or warnings.

📈

Farm Stats

Deeper analytics — daily trends, action breakdowns, charts over time. Spot problems early.

👥

Accounts Page

Check individual accounts — their last action time, daily totals, current status.

📱

Device Manager

Verify all devices are online. Offline device = idle accounts.

💡 Daily routine

Get in the habit of checking the dashboard at least twice a day — once in the morning to make sure everything started, and once in the evening to review results.

Stopping

  • Stop everything: Click "Stop Farm" on the dashboard home page
  • Stop one account: Go to Accounts → click the account → Stop
  • Stop a device group: In Device Manager, select a group → Stop All

Accounts finish their current action before stopping — they don't cut off mid-interaction.

👤 Profile Automation

Set up your slave accounts' identities in bulk from the Profile Automation page. Two modes available:

Mode 1: Mother Account Clones

Copy a mother account's style to its slaves. Great when slaves should be visibly connected to the mother (fan pages, promo accounts).

Select a mother account

In Profile Automation, choose the mother you want to clone from.

Select slave accounts

Pick which slaves should get the mother's style — select by tag, device group, or individually.

Choose what to clone

Bio template, profile picture style, username pattern, bio link — toggle each one on or off.

Preview & Apply

Review how each account will look, then click "Apply Changes". Hydra pushes the updates to each device.

Mode 2: Unique Personas

Generate completely unique, realistic identities. Ideal when slaves should look like unrelated, real people.

What gets generated:

  • Usernames — natural-looking names (e.g. "sarah.miller.23", "jake_adventures")
  • Bios — AI-generated, unique per account
  • Profile pictures — from the profile picture library
  • Full names — realistic first + last name combinations

Select accounts

Choose the accounts that need personas.

Click "Generate Personas"

Set the style (casual, professional, mixed) and any keywords to include. Click Generate.

Review & Apply

Preview each generated persona. Swap out anything you don't like, then apply.

⚠️ Don't change everything at once on new accounts

Changing username, bio, and profile picture simultaneously on a fresh account can trigger Instagram. Change one thing at a time, a day apart.

📋 Job Orders

Job Orders let you create specific, targeted tasks and distribute them across your accounts. Think of them as one-off missions on top of the regular bot routine.

Creating a Job

Go to Job Orders → New Job

Open the Job Orders page and click "New Job".

Choose the action type

Select what the job does: Follow, Like, Comment, or Report.

Add targets

Depending on the type:
• Follow: paste a list of usernames (one per line)
• Like: paste post URLs or target usernames
• Comment: paste post URLs + provide comment templates
• Report: enter the account or post to report

Assign worker accounts

Choose which accounts should work on this job — by tag, device group, or pick individually.

Set distribution & launch

Choose how to split targets across workers (evenly or all-to-all). Set daily limits per worker. Click "Create Job".

Monitoring Jobs

Active jobs show on the Job Orders page with:

  • Progress bar — targets completed out of total
  • Active workers — how many accounts are currently on the job
  • Success / Failed count
  • Estimated time remaining
[Screenshot: Job Orders page — active follow job with progress bar, worker count, success/fail stats]

Quick Mass Report

For quickly reporting an account or post:

  1. Go to Job Orders → Quick Mass Report
  2. Enter the target username or post URL
  3. Select the report reason
  4. Choose how many accounts to use
  5. Click "Launch"
🚨 Use responsibly

Mass reporting is powerful. Only use it for legitimate purposes — abuse can get your own accounts flagged.

📸 Content Management

Schedule and manage posts, reels, and stories from the Content & Media page.

Scheduling a Post

Upload media

Click "Upload" and add your photos or videos to the content library. Supported: JPG, PNG, MP4.

Create a scheduled post

Select the media, write a caption (or use AI to generate one), choose the type (feed post, reel, or story), and pick a date/time.

Assign to accounts

Choose which accounts should post this content. You can post the same content to multiple accounts or create variations.

Done

At the scheduled time, Hydra posts the content to the assigned accounts automatically.

Content Library

All uploaded media lives in the content library. You can:

  • Organize into folders
  • Tag media for easy filtering
  • Reuse across multiple accounts and schedules
  • Preview before scheduling
💡 Vary your content

Posting the exact same photo from many accounts looks suspicious. Use different crops, filters, or captions for each account when possible.

🔧 Troubleshooting

Common issues and how to fix them — all from the dashboard.

Account Shows "login_failed"

Check credentials

Go to Accounts → click the account → verify username and password are correct. Update if needed.

Retry login

Go to Login Automation → select the account → Start Login.

If it fails again

Wait 1-2 hours (Instagram may be rate-limiting), then try once more. If the password is definitely correct and it still fails, the account may need a password reset through Instagram.

Device Shows "Offline"

  • Check the Device Manager — is the phone actually powered on and connected to WiFi?
  • If you see 🔴 Offline, the phone has lost its network connection. Contact your admin to reconnect it.
  • Any accounts assigned to an offline device will be idle until the device comes back online.

Bot Is Active But Not Doing Anything

Possible Cause Check
Outside time window Go to Bot Settings → check the account's active hours
Daily limit reached Check Farm Stats or the account's action count for today
No sources configured Go to Sources & Targets — the account needs at least one source
Account not assigned to device Check in Accounts that a device + clone slot is assigned
Device offline Check Device Manager for 🔴 status

Account Got Banned

  1. The account will automatically be marked as banned in the dashboard
  2. Try to appeal: Open Instagram on the device (via Device Manager) and follow the appeal prompts
  3. If the appeal fails: Remove the account from its device slot and assign a fresh account in its place
  4. If multiple accounts got banned at once: Something is too aggressive. See Tips & Best Practices to adjust your settings.
🚨 5+ bans in one day = pause everything

If you see more than 5 accounts banned in a single day, stop the farm immediately. Review your daily limits, delays, and sources before resuming. Something needs to be dialled back.

Dashboard Won't Load

  • Try refreshing the page (Ctrl+F5 for a hard refresh)
  • Try a different browser
  • If the page still won't load, the dashboard server may need a restart — contact your admin

Other Quick Fixes

Issue Fix
Actions counted but nothing happening on the phone The phone screen may have timed out. Ask your admin to enable "Stay Awake" on the device.
Scheduled content didn't post Check that the target device was online at the scheduled time. Reschedule if needed.
Farm Stats look empty today Check if the farm is actually running (Dashboard Home → running accounts count).
Account stuck on "cooldown" Wait it out — cooldowns resolve in 24-48 hours. Don't try to force actions.

💡 Tips & Best Practices

Follow these guidelines to maximize growth and minimize account losses.

Warming Up New Accounts

New accounts need a warmup period before they can handle full activity. Start low and increase gradually:

Period Follows/Day Likes/Day Comments/Day
Day 1–3 5–10 10–20 0–3
Day 4–7 10–20 20–40 3–5
Week 2 20–40 40–80 5–10
Week 3–4 40–80 80–150 10–20
Month 2+ 80–150 150–300 20–40

Update these limits in Bot Settings for each account as you progress through the warmup.

💡 The golden rule

When in doubt, do less. It's always better to be conservative and keep an account alive than to push limits and lose it. Accounts are your most valuable resource.

Safe Daily Limits (Warmed-Up Accounts)

These are generally safe maximums for aged, fully warmed accounts:

  • Follows: 100–150/day (spread over 8–12 active hours)
  • Unfollows: 100–150/day
  • Likes: 200–300/day
  • Comments: 30–50/day
  • DMs: 20–40/day
  • Story views: 200–500/day
⚠️ Limits change over time

Instagram constantly adjusts their detection. What's safe today may be too much next month. Watch your ban rate — if it goes up, reduce limits by 20%.

Avoiding Bans

  • Use realistic delays — 30–120 seconds between actions. No one likes 50 posts in 2 minutes.
  • Mix your actions — don't just follow 100 people. Alternate follows with likes, reels, and comments.
  • Take breaks — enable session breaks in Bot Settings. Real people don't scroll for 12 hours straight.
  • Post content — accounts that only follow/like but never post look suspicious.
  • Complete profiles first — no profile picture + no bio + no posts = faster bans.
  • Warm up new accounts — always follow the schedule above.
  • Unfollow gradually — don't mass-unfollow. Spread it over days.

Tag Strategy

One Tag Per Mother Account

Growing 3 mothers? Create 3 tags. All slaves for Mother A share "tag_a", etc. This ensures the dedup system keeps your farm looking natural.

Why It Matters

  • Without dedup: @john gets followed by 15 of your accounts in one day → looks fake → reports → bans
  • With dedup: @john gets followed by 1 account → natural → no flags

General Tips

  • Check the dashboard daily — catch problems early before they cascade
  • Watch Farm Stats trends — a sudden drop in daily actions means something is wrong
  • Rotate comment templates — stale, repetitive comments get flagged. Update them regularly.
  • Don't start all accounts simultaneously — stagger start times across the time window to spread activity naturally
  • Ask your admin for regular backups — your account database is irreplaceable
