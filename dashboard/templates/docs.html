Story Mention Sticker

New Story Editor share_to_story.py

Automatically adds an @mention sticker to shared stories via the Instagram sticker picker. When sharing a source account's post/reel to your story, the bot opens the sticker picker, searches for "mention", selects the MENTION sticker, types the source username, picks the suggestion, and confirms with Done.

Why it matters: Mention stickers increase engagement and visibility for the source account, which can drive reciprocal engagement. The bot handles the full sticker picker flow including search, selection, and suggestion matching.

Navigation Flow
Step 1
Story editor loaded → swipe up to open sticker picker
Step 2
Type mention in sticker search bar
Step 3
Tap MENTION sticker from search results
Step 4
Type target username → select from suggestions
Step 5
Tap Done → sticker placed on story
Settings
# Account settings JSON "story_mention_enabled": true, "story_mention_target": "source_username" # defaults to source if empty
Key Selectors
ElementSelector
Sticker picker openswipe up OR d(description="Emojis and stickers")
Search bard(className="android.widget.EditText")
MENTION stickerd(description="Mention Sticker") / d(text="MENTION")
Confirmd(text="Done")

Story Link Sticker

New Story Editor share_to_story.py

Adds a link sticker to shared stories via the sticker picker. Opens the sticker picker, searches "link", selects the LINK sticker, enters the configured URL in the form's first EditText field, and confirms with Done. Supports any URL including affiliate links, landing pages, or profile links.

Live-tested on: IG clone com.instagram.androie (2026-02-21). The link sticker appears as a blue "LINK" button in search results under the "Stickers" heading.

Navigation Flow
Step 1
Story editor loaded → swipe up to open sticker picker
Step 2
Type link in sticker search bar → wait 3s for results
Step 3
Tap LINK sticker (blue button) from search results
Step 4
Enter URL in first EditText field (URL field)
Step 5
Tap Done → link sticker placed on story
Settings
# Account settings JSON "story_link_sticker_enabled": true, "story_link_sticker_url": "https://example.com/landing"
Key Selectors

Business Profile Switch

Business switch_to_business.py

Automates the full Personal → Professional (Business) account conversion. Navigates through the Settings UI: Profile → Options hamburger → "Settings and activity" → scroll to "Account type and tools" → "Switch to professional account" → complete the multi-step setup wizard (intro, category selection, contact info, Facebook skip, setup dismissal).

Clone-specific: Menu item is "Settings and activity" (NOT "Settings and privacy"). Needs 6+ swipe-scrolls to reach "Account type and tools". Uses clone-specific bb_primary_action resource ID for buttons. Category picker uses RadioButtons.

Navigation Flow

Step 2

Step 3

Step 4
Category picker: select from SAFE_CATEGORIES list via RadioButton
Step 5
Contact info → Next | Skip Facebook connect
Step 6
Dismiss setup complete screen via Close (X) button
Step 7
Verify: check for "Professional dashboard" or "Insights" on profile

Instagram Insights Scraper

New Analytics scrape_insights.py

Automated Professional Dashboard analytics scraper. Navigates from Profile through each insights section (Views, Interactions, Followers) and extracts all available metrics via dump_hierarchy() + regex parsing. Supports both a quick overview scrape and full drill-down with detailed breakdowns.

Two modes:

  • scrape_insights_overview(d) — Dashboard overview only (Views, Interactions, New followers, Content shared). Fast, ~10 seconds.
  • scrape_insights(d) — Full drill-down into each section with content type breakdown, demographics, comparison periods. ~45 seconds.
Full Navigation Flow
Step 1
Profile tab → "Professional dashboard" → scrape overview
Step 2
Tap "Views" → scrape total, follower/non-follower split, reached, content type breakdown → scroll for profile visits, link taps → back
Step 3
Tap "Interactions" → scrape total, follower split, likes, content type → back
Step 4
Tap "New followers" → scrape total, check if ≥100 for demographics → scroll for cities, countries, age, gender → back
Step 5
Return complete data dict → optionally save to account_insights_v2 table
Data Extraction Techniques
# Total from content-desc: "153 Views" re.search(r'content-desc="([\d,]+)\s*Views?"', xml) # Percentage near label: finds "6%" after "Followers" _extract_percentage(xml, "Followers") # Value near label: finds "115" after "Accounts reached" _extract_value_near_label(xml, "Accounts reached") # Date range: "Jan 22 - Feb 20" re.search(r'text="([A-Z][a-z]{2}\s+\d{1,2}\s*[-–]\s*[A-Z][a-z]{2}\s+\d{1,2})"', xml)
Return Data Structure
{ 'scraped_at': '2026-02-22T01:00:00', 'date_range': 'Jan 22 - Feb 20', 'overview': {'views': 153, 'interactions': 6, 'new_followers': 0, 'content_shared': 1}, 'views_detail': { 'total': 153, 'followers_pct': 6.0, 'non_followers_pct': 94.0, 'accounts_reached': 115, 'profile_visits': 12, 'content_type': {'reels_pct': 99.3, 'posts_pct': 0.7}, ... }, 'interactions_detail': {'total': 6, 'likes': None, ...}, 'followers_detail': {'total': 29, 'demographics_available': False, ...} }
Usage
from automation.actions.scrape_insights import scrape_insights, save_insights_to_db # Full scrape data = scrape_insights(d, pkg='com.instagram.androie') # Save to DB save_insights_to_db(data, account_id='myaccount', device_serial='192.168.1.100:5555') # Quick overview only (no drill-down) overview = scrape_insights_overview(d)

Tag-Based Follow Dedup v2

New Cross-Device tag_based_automation.py

Cross-device deduplication for tag-based follow campaigns. The tag_followed_targets table tracks every username followed across all devices and accounts, preventing the same target from being followed by multiple farm accounts. Each record stores the account that did the follow, which device, and when — enabling farm-wide dedup queries.

Before following a target, the bot checks:

SELECT 1 FROM tag_followed_targets WHERE target_username = ? AND followed_at > datetime('now', '-30 days') LIMIT 1

If the target was followed by any account in the last 30 days, it's skipped. This prevents wasted follows and reduces the chance of Instagram detecting coordinated behavior.

Dedup Table Schema

Cross-Device Flow
Step 1
Bot picks target from follower list of source account
Step 2
Query tag_followed_targets for target_username across ALL devices
Step 3
If found → skip target, move to next
Step 4
If not found → follow target → INSERT into dedup table
Step 5
All devices share the same dedup table → no duplicates
